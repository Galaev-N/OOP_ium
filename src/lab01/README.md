# Предметная область -> Интернет магазин
## Что решил реализовывать? Продукт на каком-то сайте, скажем заказ продуктов из магазины по интернету

Отлично, решил реализовывать продукт на сайте

## Характеристики продукта на интернет магазине
Название - чтобы что-то продать, нужно придумать этому имя, тип данных - str

Цена - алло, бизнес? да-да деньги, сколько стоит товар? Цена, тип данных - int

Количество - должно быть определено, чтобы мы не могли оформить заказ продукта, которого неот в наличии, тип данных - int

Изготовитель/продавец - кто-то должен нести ответственность и выслушивать жалобы? У него будет имя, тип данных - str

СебеСтоимость - не обязательно, но все же прусть будет на всякий случай, сделаю приватным атрибутом атрибутом, тип данных - float

Айди - характеристика для системы, чтобы она могла понимать с чем мы имеем дело - str 

Описание - опционально по задумке может быть добавлено изготовителем - str

Комментарии - комментарии пользователей к продукту на странице инет магазина - dict из str:str

Оценка - ну тут совсем зашквар, вдруг вам не понравилось или наоборот - float

Статус - честно, от балды делал, отображает находиться ли продукт в дефиците - int, защищенный атрибут

# А что мы можем делать с продуктом или что он может делать?
## С самим продуктом я реализовал 4 опции:
1. Заказ(бизнес метод) - ибо че за страница, если нельзя заказать продукт. Нужно кол-во денег в кошельке 

2. Отзыв(бизнес метод) - ну вот и до хейтеров добрались, добавляет в dict комментариев новый, под стандартным ником

3. Оценить(property) - устанавливат для продукта оценку, которую вы ему указали(глупо реализовано, но все же)

4. Инфо(property) - удобно выведем всю поднаготную про продукт

# Инварианты:
Название: str, не пустая

Цена: int, > 0

Количество: int, >= 0

Продавец/изготавитель: str, не пустая

СебеСтоимость: float, >= 0.0

Айди: str, начинается с #, длинна - 7

Описание: str, не пустая

Комментарии: dict

Оценка: float. >= 0.0 и <= 5.0

Статус: bool/int(0 или 1)

# Магические методы

str - выведет краткую инфу по продукту, без приватных и защищенных атрибутов

repr - вывод информации для разраба, со всеми атрибутами

eq - сравнивает продукты исключительно по названию, глупо, но просто
